# IVLTI

一个基于《第五人格》IVL 赛事圈层梗文化制作的网页人格测试项目。

## 在线地址

- 已部署上线：[ivlti.asia](https://ivlti.asia)
- 小红书链接：[http://xhslink.com/o/5fqLQi5icIP](http://xhslink.com/o/5fqLQi5icIP)

用户通过 30 道题得到 10 个维度分数，最终匹配到一个最接近的 IVL 角色类型。项目为纯前端静态页面，不依赖后端，打开 `index.html` 即可运行。

## 项目特点

- 纯 HTML / CSS / JavaScript 实现，部署和分享都很轻量
- 首页、答题页、结果页三段式流程
- 30 题单页翻题模式，带进度条与上一题 / 下一题导航
- 22 个角色结果，包含角色文案、向量、图片映射
- 根据 10 个维度做曼哈顿距离匹配，输出最接近角色
- 页面风格偏浅金暖色，适合粉圈测试和二创传播

## 当前测试内容

- 题目数量：30 题
- 维度数量：10 维

## 结果计算逻辑

项目当前的匹配逻辑写在 `index.html` 中：

1. 用户完成 30 道题
2. 每 3 题汇总为一个维度分数
3. 形成 10 维用户向量
4. 与 `NORMAL_TYPES` 中每个角色的 10 维向量做距离计算
5. 选出距离最小的角色作为结果

当前使用的是简单直接的绝对差累加方案，适合轻量测试场景，也方便后续手动调参。

## 可调整内容

### 1. 修改题目

在 `index.html` 里的 `questions` 数组中修改题目内容：

- 每道题有 `id`、`dim`、`text`、`options`
- 每个维度目前固定 3 题

### 2. 修改角色结果

角色相关数据主要有三部分：

- `TYPE_LIBRARY`：角色名称、简介、结果文案
- `TYPE_IMAGES`：角色图片映射
- `NORMAL_TYPES`：角色 10 维向量

如果你要新增角色，通常需要同时补这三处。

### 3. 调整角色爆率

如果某个角色太容易或太难测出，优先调整它在 `NORMAL_TYPES` 里的向量。

建议原则：

- 爆率过高：把角色从大众区间移开，增强 1 到 2 个核心特征
- 爆率过低：把过低的非核心维度稍微往中间拉近
- 每次只改少量维度，方便观察变化
